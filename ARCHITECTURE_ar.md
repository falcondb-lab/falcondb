## 0. الخاصية الأساسية: ضمان الالتزام الحتمي (DCG)

> **إذا أعاد FalconDB "committed"، فإن المعاملة تنجو من أي عطل في عقدة واحدة، وأي تجاوز للأعطال، وأي عملية استرداد — بدون استثناء.**

يتم فرض **ضمان الالتزام الحتمي** بواسطة نموذج نقطة الالتزام (§1 في `falcon_common::consistency`): تحت سياسة الالتزام النشطة، لا يتم إرسال أي ACK للعميل قبل استمرارية WAL. مراحل الالتزام الأربع — **WAL مسجل → WAL مُستمر → مرئي → مؤكد** — تتقدم بشكل صارم للأمام؛ أي محاولة تراجع تُرفض كـ `InvariantViolation`.

- الأدلة: [docs/consistency_evidence_map.md](docs/consistency_evidence_map.md)
- مخطط التسلسل: [docs/commit_sequence.md](docs/commit_sequence.md)
- غير الأهداف: [docs/non_goals.md](docs/non_goals.md)

## 1. نظرة عامة على البنية

### 1.1 تدفق البيانات (المسار العادي: SELECT)

```
psql ──TCP──▶ [طبقة البروتوكول (PG Wire)]
                  │
                  ▼
            [واجهة SQL: Parse → Bind → Analyze]
                  │
                  ▼
            [المخطط / الموجه]
                  │
          ┌───────┴────────┐
          │ شريحة واحدة   │ شرائح متعددة
          ▼                ▼
    [المنفذ المحلي]    [المنسق الموزع]
          │                │
          ▼                ▼
    [مدير المعاملات ─── MVCC ─── محرك التخزين]
          │
          ▼
    [جدول الذاكرة + الفهرس]
          │
          ▼ (غير متزامن)
    [WAL / نقطة التحقق]
```

### 1.2 تدفق البيانات (مسار الكتابة)

```
عميل INSERT ──▶ PG Wire ──▶ واجهة SQL ──▶ المخطط/الموجه
    ──▶ بدء المعاملة ──▶ تحديد الشريحة (تجزئة PK)
    ──▶ LocalTxn (مسار سريع، شريحة واحدة، بدون 2PC)
        │  أو GlobalTxn (مسار بطيء، شرائح متعددة، XA-2PC)
    ──▶ التزام ──▶ تطبيق MemTable + WAL fsync
    ──▶ إرسال WAL إلى النسخ المتماثلة (غير متزامن، مخطط A)
    ──▶ تأكيد العميل
```

### 1.3 أدوار العقد (M1: أساسي / نسخة متماثلة)

يستخدم M1 طوبولوجيا أساسي-نسخة متماثلة لكل شريحة:
- **الأساسي**: يقبل القراءة/الكتابة؛ يؤكد الالتزام بعد استمرارية WAL.
- **النسخة المتماثلة**: تستقبل قطع WAL؛ للقراءة فقط قبل الترقية.
- **الترقية**: تجاوز أعطال قائم على السياج (fence → اللحاق → التبديل → unfence).

## 2. قائمة الوحدات

### 2.1 `protocol_pg` — بروتوكول PostgreSQL Wire

| العنصر | التفاصيل |
|--------|----------|
| **المسؤولية** | قبول TCP، مصافحة PG، تحليل Simple/Extended Query، تسلسل الاستجابات |
| **الأنواع الرئيسية** | `PgConnection`, `PgMessage`, `AuthMethod`, `PgSession` |
| **إعادة استخدام OSS** | crate `pgwire` (MIT) لترميز الرسائل |

### 2.2 `sql_frontend` — المحلل / المحلل الدلالي / الرابط

| العنصر | التفاصيل |
|--------|----------|
| **المسؤولية** | نص SQL → AST → خطة منطقية محلولة مع مراجع الكتالوج |
| **الأنواع الرئيسية** | `Statement`, `Expr`, `TableRef`, `ColumnRef`, `BoundStatement` |
| **إعادة استخدام OSS** | `sqlparser-rs` (Apache-2.0) |

### 2.3 `planner_router` — مخطط الاستعلام والموجه الموزع

| العنصر | التفاصيل |
|--------|----------|
| **المسؤولية** | خطة منطقية → خطة فيزيائية؛ تحديد الشريحة المستهدفة؛ دفع المرشحات |
| **الأنواع الرئيسية** | `LogicalPlan`, `PhysicalPlan`, `ShardTarget`, `PushDown` |

### 2.4 `executor` — محرك التنفيذ

| العنصر | التفاصيل |
|--------|----------|
| **المسؤولية** | تنفيذ عوامل الخطة الفيزيائية؛ حاكم الاستعلام؛ التجميع المدمج المتدفق |
| **الأنواع الرئيسية** | `Operator` trait, `SeqScan`, `IndexScan`, `Filter`, `Project`, `Sort`, `Agg`, `QueryGovernor` |
| **المسار السريع** | `exec_fused_aggregate` — WHERE+GROUP BY+تجميع في مرور واحد MVCC (صفر نسخ صفوف) |

### 2.5 `txn` — مدير المعاملات

| العنصر | التفاصيل |
|--------|----------|
| **المسؤولية** | بدء/التزام/إلغاء المعاملات؛ تعيين طوابع MVCC الزمنية؛ التحقق OCC (SI) |
| **الأنواع الرئيسية** | `TxnManager`, `TxnHandle`, `TxnId`, `Timestamp`, `IsolationLevel` |

### 2.6 `storage` — محرك التخزين في الذاكرة

| العنصر | التفاصيل |
|--------|----------|
| **المسؤولية** | تخزين الصفوف وإصدارات MVCC في الذاكرة؛ صيانة الفهارس؛ كتابة WAL + الاسترداد؛ GC MVCC |
| **الأنواع الرئيسية** | `StorageEngine`, `MemTable`, `VersionChain`, `SecondaryIndex`, `WalWriter`, `GcRunner` |

### 2.7 `raft` — الإجماع (بذرة — خارج مسار الإنتاج)

> **⚠️ هذا الـ crate هو بذرة عقدة واحدة. لا يُستخدم في مسار بيانات الإنتاج.**

### 2.8 `cluster` — المجموعة والنسخ المتماثل والتنفيذ الموزع

| العنصر | التفاصيل |
|--------|----------|
| **المسؤولية** | خريطة الشرائح؛ النسخ المتماثل أساسي-نسخة بناءً على WAL؛ الترقية/التجاوز مع السياج؛ التنفيذ الموزع scatter/gather |
| **الأنواع الرئيسية** | `ShardMap`, `WalChunk`, `ReplicationTransport`, `DistributedQueryEngine` |

### 2.9 `observability` — المقاييس / التتبع / التسجيل

مقاييس Prometheus، تتبع OpenTelemetry، تسجيل منظم.

### 2.10 `common` — البنية التحتية المشتركة

أنواع مشتركة، تسلسل الأخطاء، التكوين، نظام أنواع PG، تمثيل datum، RBAC.

## 3. نطاق MVP

### 3.1 مدعوم

| الفئة | النطاق |
|-------|--------|
| **البروتوكول** | PG wire Simple Query + Extended Query؛ مصادقة trust |
| **SQL DDL** | `CREATE TABLE`, `DROP TABLE`, `TRUNCATE`, `ALTER TABLE`, `CREATE INDEX`, `CREATE VIEW` |
| **SQL DML** | `INSERT`, `UPDATE`, `DELETE` (مع `RETURNING`)، `SELECT`، `JOIN`، استعلامات فرعية، CTE، CTE تكراري، دوال النافذة، 260+ دالة عددية |
| **الأنواع** | `INT`, `BIGINT`, `TEXT`, `BOOL`, `FLOAT8`/`NUMERIC`/`DECIMAL`, `TIMESTAMP`, `JSONB`, `ARRAY`, `DATE` |
| **المعاملات** | `BEGIN`, `COMMIT`, `ROLLBACK`؛ مستوى العزل Read Committed |
| **التخزين** | مخزن صفوف في الذاكرة؛ فهرس تجزئة PK؛ فهرس ثانوي BTree |
| **الاستمرارية** | WAL (شرائح 64 ميجابايت، التزام جماعي)؛ الاسترداد بعد الأعطال |
| **الموزع** | تقسيم بتجزئة PK؛ نسخ WAL عبر gRPC؛ مسار سريع شريحة واحدة + 2PC عبر الشرائح |

### 3.2 غير مدعوم (M1)

- النسخ المنطقي، CDC
- تغيير المخطط عبر الإنترنت
- تعدد المستأجرين
- إعادة التوازن التلقائي (يدوي فقط)

## 5. تصميم المعاملات والاتساق

### 5.2 مُنفّذ: المسار السريع + المسار البطيء + OCC

- **LocalTxn (المسار السريع)**: معاملة شريحة واحدة، التزام بالتحقق OCC تحت عزل اللقطة. بدون حمل 2PC.
- **GlobalTxn (المسار البطيء)**: معاملة عبر الشرائح مع XA-2PC.
- **فرض الثوابت الصارمة**: `TxnContext.validate_commit_invariants()` يُرجع خطأ `InvariantViolation`.

### 5.4 GC MVCC

- **نقطة الأمان**: `gc_safepoint = min(min_active_ts, replica_safe_ts) - 1`
- **واعي بـ WAL**: لا يجمع أبداً الإصدارات غير الملتزمة أو الملغاة
- **آمن للنسخ المتماثل**: يحترم الطوابع الزمنية المطبقة بواسطة النسخ المتماثلة

### 5.5 النسخ المتماثل — نقل WAL عبر gRPC

```
الأساسي ──قطع WAL──▶ النسخة المتماثلة
         (بث gRPC, tonic, RPC SubscribeWal)
```

> **⚠️ حد صارم**: crate `falcon_raft` هو بذرة عقدة واحدة. نسخ نقل WAL المتماثلة **ليست** متابعين Raft.
